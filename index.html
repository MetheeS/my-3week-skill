05 — Debugging in depth

One symptom. Four bugs.

The azure-costdown history's clearest exhibit: a single wrong resource count unfolded into four distinct bugs, fixed in sequence — each fix exposing the next.

1
Multi-line KQL through cmd.exe dropped a flag
FIX → flatten the query to a single line
↓
2
Nested double quotes broke shell parsing
FIX → swap KQL literals to single quotes
↓
3
CLI returned snake_case keys, not the API's camelCase
FIX → tolerate both naming conventions
↓
4
nextLink pagination needed the full body every POST
FIX → stop assuming GET-state-encoding

What makes this senior-level isn't that the bugs were found. It's that you ran the CLI manually, outside Python, to inspect the real output shape before editing code — moving from "fix the code" to "fix my model of how the system behaves."

06 — Steering the tool

The skill that separates the rest.

Three moments from the record where you didn't follow the AI — you directed it.

override
When Claude built elaborate join workarounds against the warehouse, you cut it off: "just use simple query." You recognised over-engineering and were right.
audit
The i-facebook Phase 0 research caught five technical errors in Claude's own sprint plan — before any of them reached code.
"Question the model" — look at the actual SDK source, the actual error, the actual runtime, rather than trusting a confident-sounding answer.
— a standing habit, named in your own i-facebook history. Keep it. It's your single most valuable working habit.
07 — What's next

Honest gaps — none of them serious.

Every item here is a consistency problem, not a capability one. You've already shown each missing habit at least once.

GAP 01
Verification is reactive
A wrong assumption causes a bug, the bug costs time, then the habit is adopted. Action — inspect the real data shape before writing the code that consumes it. Make it a checklist step, not a re-learned lesson.
GAP 02
Testing rigour is uneven
Photo Booth has a real three-layer suite; Canteen explicitly has none. Action — make the Photo Booth approach your baseline and carry it into the next project from day one.
GAP 03
The same bug class recurs
Windows→Linux path and encoding issues appear in three projects, each codified per-repo. Action — keep one cross-project gotchas file or reusable skill. You're 80% there.
GAP 04
Operational loose ends
A documented maxReplicas drift; an unfinished polling path. Action — when a history says "known issue," ticket it. Written-but-untracked tends to stay unfixed.
